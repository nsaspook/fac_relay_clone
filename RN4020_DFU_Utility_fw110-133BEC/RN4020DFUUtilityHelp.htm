RN4020 DFU Utility Help

 

Overview

 

The RN4020 DFU Utility can be used to perform Device Firmware Update (DFU) on the RN4020 Bluetooth Low Energy module using a wired connection over Universal Asynchronous Receiver/Transmitter (UART) interface. This utility can be used for both RN4020 firmware upgrade and downgrade needs. To obtain the most recent and complete RN4020 data sheet, user’s guide, visit the Microchip web site: www.microchip.com/rn4020.

 

 

WARNING:

-       While the DFU is in progress, RN4020 should not be disconnected and DFU process should not be interrupted. If the DFU process is interrupted the RN4020 module might end up bricked.

-       Use Microchip released valid RN4020 DFU (.bin) file only. Using an invalid DFU (.bin) file will result in a bricked module.

-       This utility cannot be used to un-brick the module.

-       If the module gets bricked please contact a Microchip Sales Representative to get the issue resolved.

 

Operation

 

1)     Connect the RN4020 PICtail to a Windows PC using the USB mini-B cable.

 

2)     Wait for the RN4020 PICtail to enumerate as a Serial Port device successfully.

 

3)     After the RN4020 PICtail enumerates as a Serial Port device, open the Device Manager on Windows PC and note down the COM port number assigned to the RN4020 PICtail under ‘Ports’.

 

4)     Run the RN4020 DFU Utility (RN4020DFUUtility.exe) on the Windows PC.

 

5)     Ensure that the RN4020 PICtail is still connected to the Windows PC and successfully enumerated. Click ‘Scan’ button to scan for all the UART COM ports available on the Windows PC.

Note: Ensure that the COM port on which the RN4020 PICtail is enumerated is not already open by another application like a Serial Terminal application.

 

6)     Click on the drop down menu of the UART COM Port and select the correct COM port which is assigned to the RN4020 PICtail to be firmware updated.

 

7)     A set of DFU files for the released versions of the RN4020 firmware are provided. Choose the desired DFU file from the DFU File drop down menu.

OR

Check the ‘Browse DFU File’ checkbox and click ‘Browse’ button to browse and select a valid DFU (.bin) file.

Note: Use Microchip released valid RN4020 DFU (.bin) file only. Using an invalid DFU (.bin) file will result in a bricked module.

 

8)     With the RN4020 PICtail still connected, COM port and valid DFU file selected, click ‘Update’ button to start the RN4020 DFU update process. The progress bar shows the progress of the DFU process.

 

9)     After the DFU process is complete, an information dialog window pops up to inform a successful DFU.

Note: After the DFU is successful, the RN4020 will be in factory defaults state.
